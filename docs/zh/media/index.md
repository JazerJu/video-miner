# 媒体凭据

媒体凭据用于配置下载平台登录信息、浏览器 Cookie、统一代理和 yt-dlp 依赖状态。这里的配置会影响 B站、YouTube、Podcast 下载，以及 LLM 相关代理开关。

## B站登录凭据

| 设置 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| B站登录 SESSDATA | 密码输入 | 空 | 用于登录 B站获取高清视频和字幕。 |
| 复制 | 按钮 | 无 | 复制当前输入的 SESSDATA。 |
| 保存并验证 | 按钮 | 无 | 保存并验证 B站登录态，成功时显示账号、UID 和过期时间。 |
| 登录态状态 | 状态提示 | 未配置 | 显示未配置、已配置、有效、无效或已过期。 |

## YouTube cookies.txt

| 设置 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| cookies.txt 上传 | 文件上传 | 未上传 | 上传从浏览器导出的 YouTube `cookies.txt`，用于访问需要登录的视频。 |
| 覆盖上传 | 文件上传 | 无 | 已存在 cookies.txt 时，上传新文件会覆盖旧文件。 |
| 文件信息 | 悬停提示 | 无 | 显示 cookies.txt 修改时间和文件大小。 |

## 网络代理设置

| 设置 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| 代理服务器地址 | 文本输入 | 空 | 统一代理地址，例如 `http://host.docker.internal:7890`。留空则不使用代理。 |
| 流媒体下载走代理 | 复选框 | 关闭 | YouTube 和 Podcast 下载使用代理。 |
| B站下载走代理 | 复选框 | 关闭 | B站下载使用代理。 |
| 字幕分割 LLM 走代理 | 复选框 | 关闭 | 断句 LLM 请求使用代理。 |
| 字幕翻译 LLM 走代理 | 复选框 | 关闭 | 翻译 LLM 请求使用代理。 |

## yt-dlp 管理

| 设置 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| yt-dlp 版本 | 状态显示 | 后端检测值 | 显示当前 yt-dlp 版本。 |
| EJS 脚本 | 状态显示 | 后端检测值 | 显示 yt-dlp EJS 支持状态。 |
| Node.js 运行时 | 状态显示 | 后端检测值 | 显示 Node.js 是否可用和版本。 |
| Node 要求 | 状态显示 | 后端检测值 | 显示所需 Node.js 版本。 |
| 安装 yt-dlp 依赖 | 按钮 | 无 | 安装 `yt-dlp[default]`、`yt-dlp-ejs`、`pycryptodomex`、`brotli` 等依赖。 |
| 检测升级 yt-dlp | 按钮 | 无 | 检查最新版本并升级 yt-dlp 及依赖。 |
| 刷新状态 | 按钮 | 无 | 重新读取 yt-dlp 管理状态。 |

## 如何获取 B站 SESSDATA

1. 在浏览器中登录 B站。
2. 打开浏览器开发者工具。
3. 在应用或存储面板中找到 B站 Cookie。
4. 复制名为 `SESSDATA` 的值。
5. 粘贴到设置面板并点击保存并验证。

## 如何准备 YouTube cookies.txt

1. 在浏览器中登录 YouTube。
2. 安装浏览器扩展 Get cookies.txt LOCALLY。
3. 导出 `cookies.txt`。
4. 在媒体凭据页上传该文件。

> Docker 部署也可以通过 `HTTPS_PROXY` 环境变量配置代理，但设置面板中的代理地址会作为应用内统一代理使用。
